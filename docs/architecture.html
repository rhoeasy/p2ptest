p2ptest — DDD Architecture

Three bounded contexts + event bus + coordinator + Ed25519 identity

Bounded Contexts

  • Discovery — network topology queries
  • Membership — peer relationship lifecycle
  • Messaging — bidirectional message stream
  • No cross-imports; Node coordinates

Event Flow

  • Membership.Emit → Notifier → CLI/Web
  • Node subscribes to peer_discovered
  • Side-effect chain: discovered → broadcastJoin
  • Loop break: registry idempotency check

Ed25519 Security

  • Node generates identity at startup
  • PublicKey in NodeID (proto field)
  • Handshake/Heartbeat/Envelope signed
  • Backward-compatible: unsigned passes
